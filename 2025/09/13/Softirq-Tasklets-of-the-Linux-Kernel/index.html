Introduction for Softirq and Tasklets of the Linux kernel

1 BG

计算机设备中，外设的速度是远低于cpu的计算速度的，一般采用中断这种异步方式来进行时间通知。

在整个中断处理过程中，arm64 的 CPU 全局中断是自动 disable 的 (PSTATE 寄存器中的 interrupt bit 被 masks)。如果用户想支持 interrupt nested，需要自己在中断服务程序中使能中断。Linux 现在是不使用中断嵌套的。

这就需要保证中断的isr需要尽可能短，将非紧急但很占用时间的具体操作延后。即所谓的defered interrupt或中断下半部。

这里不关注Arm的中断触发到处理流程，仅记录两种defered interrupt的方式：softirq和tasklet

2 Software Interrupts

1 init

当前内核中softirq有9个，均是静态分配。

其中两个用来实现tasklet（HI_SOFTIRQ和TASKLET_SOFTIRQ），两个用于网络的发送和接收操作（NET_TX_SOFTIRQ和NET_RX_SOFTIRQ），一个用于调度器（SCHED_SOFTIRQ），实现SMP系统上周期性的负载均衡。在启用高分辨率定时器时，还需要一个HRTIMER_SOFTIRQ 还有一个用作read_copy_update。

3 Tasklets

3.1 tasklet trigger

首先是tasklet_schdule() →__tasklet_schedule() →__tasklet_schedule_common() ，将tasklet_struct添加至tasklet_vec链表中，当TASKLET_SOFTIRQ执行时，遍历该链表，执行其中所有tasklet_struct的func。

最终还是走到raise_softirq_irqoff() 来设置softirq的pending bit。

3.2 init

其中tasklet_struct 字段含义如下：

字段 作用
next 用于链接 tasklet 的链表结构（通常由调度器使用）
state 表示 tasklet 的状态，如是否正在运行或已调度
count 原子计数器，用于控制 tasklet 是否可执行（非零表示禁止执行）
func 实际执行的函数
use_callback 是否使用回调机制（通常为 false）
data 传递给 func 的参数
